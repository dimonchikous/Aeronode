# Инструкция

## Оффлайн
### Сборка устройства
Для начала нужно купить все необходимые компоненты:
* **ESP32-WROOM-32 DevKit V1** (Микроконтроллер)
* **BME280** (Измеряет температуру, влажность, давление)
* **SCD-40** (Измеритель CO2)
* **OLED-экран SSD1306** 128x64p (Белого цвета) 

<span style="color:red">**ОБРАТИТЕ ВНИМАНИЕ! ВСЕ КОМПОНЕНТЫ ДОЛЖНЫ БЫТЬ НА I2C ШИНЕ!**</span>

При покупке обратите внимание на то, как выглядит BME280, так как из-за внешнего сходства с BMP280 часто происходит путаница!
<img src="img/BMPvsBME.jpeg" alt="bme and bmp" width="50%" />

Далее необходимо осуществить подключение всех модулей в одно устройство согласно схеме:
![Схема подключения](img/connectionScheme.png)

1. **3.3V (ESP32)** -> **VIN** (BME280) -> **VDD** (SCD-40, может называться VCC) -> **VCC** (SSD1306)
2. **GND (ESP32)** -> **GND** (BME280) -> **GND** (SCD-40) -> **GND** (SSD1306)
3. **D21 (ESP32)** -> **SDA** (BME280) -> **SDA** (SCD-40) -> **SDA** (SSD1306)
4. **D22 (ESP32)** -> **SCL** (BME280) -> **SCL** (SCD-40) -> **SCL** (SSD1306)

### Прошивка

#### Способ 1: Метод PlatformIO
1. Скопируйте репозиторий:
```bash
git clone https://github.com/dimonchikous/Aeronode.git
```
2. Перейдите в каталог `Aeronode/esp32/`
3. (Необязательно) Отконфигурируйте файлы `lib/AeronodeConfig/AeronodeConfig.h` и `lib/AeroDisplay/AeroDisplay.h`
4. Соедините ESP32 с компьютером по USB.
5. Скомпилируйте и отправьте код на ESP32:
```bash
pio run -t upload
```

#### Способ 2: Метод esptool
1. Установите **Python**.
2. Установите esptool:
```bash
pip install esptool
```
3. Подключите ESP32 к компьютеру и определите порт.

**На Linux:** 
```bash
ls /dev/ttyUSB*
# или если не нашлось:
ls /dev/ttyACM*
```
Обычно это `/dev/ttyUSB0`.

**На Windows:**
* Откройте **Диспетчер устройств**.
* Загляните в раздел **"Порты (COM и LPT)"**. Обычно это `COM3`, `COM4` и т.д.

4. Скачайте файл `Aeronode.bin` из раздела **Releases**.
5. Прошейте устройство:
```bash
esptool --chip esp32 --port <ВашПорт> --baud 921600 write_flash 0x0 Aeronode.bin
```
*(Замените `<ВашПорт>` на имя вашего порта)*.

#### Способ 3: Метод Adafruit WebSerial
1. Подключите ESP32 к компьютеру. Убедитесь, что никакая другая программа не использует порт.
2. Из раздела **Releases** скачайте файл `Aeronode.bin`.
3. Откройте [Adafruit WebSerial ESPTool](https://adafruit.github.io/Adafruit_WebSerial_ESPTool/). 
4. Нажмите **Connect** и в появившемся окне выберите вашу плату (например, "USB Serial").
5. В появившемся меню нажмите на кнопку **"Choose a file"** и выберите `Aeronode-offline.bin`.
    * **Важно:** Убедитесь, что **Offset** выставлен на **`0x0`**.
6. Нажмите **Program** и дождитесь полной загрузки. После этого нажмите **Reset** на ESP32 или переподключите кабель.

---

#### Дополнительно
* Если вам не помогли способы выше, пожалуйста, создайте обращение в разделе **Issues** или попробуйте другие доступные инструменты прошивки.


## Онлайн
### Конфигурация ESP32
1. Откройте в любом текстовом редакторе файл **AeronodeConfig.h**
    * `esp32/lib/AeronodeConfig/AeronodeConfig.h`
2. Включите(выставьте на 1) параметр **AERONODE_ONLINE**
    * `#define AERONODE_ONLINE 1`
3. Сохраните файл
4. Загрузите прошивку(описано выше).
    * Если вы используете **.bin** способ, загрузите прошивку с названием Aeronode-online.bin
5. Запустив устройство впервые, подключитесь к точке доступа, которое оно создало для настройки.

### Конфигурация бота.
**Важно**: Инструкция снизу написана преимущественно для Linux-систем.
1. Установите необходимые пакеты
    * python3-venv 
    * python3-pip
    * ufw
    * rsync
2. Настройте **ufw**
```bash
sudo ufw allow 5000/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```
3. Создайте отдельного пользователя под проект и залогиньтесь от него.
4. Склонируйте проект в **домашнюю** директорию и зайдите в директорию проекта.
5. Настройте виртуальное окружение
```bash
python3 -m venv venv
source venv/bin/activate #или activate.fish в зависимости от окружения.
pip install -r requiriments.txt
```
6. Конфигурация .env
Откройте файл **.env** и настройте его.

**BOT_TOKEN** - Токен бота Telegram полученный в боте @BotFather.

**USE_WHITELIST** - Флаг использования белого списка нод. По умолчанию False.

**Пример**
```env
BOT_TOKEN=ТОКЕН_БОТА

USE_WHITELIST=False
```
7. Конфигурация users.txt
Откройте файл **users.txt**.

Вписывайте id пользователей telegram, которые могут пользоваться вашим ботом. Например:
```
000000000
```
8. Добавление нод в чёрный список.
Откройте файл **blacklist.txt**

Вписывайте последние 2 байта MAC-адреса нод, которых вы хотите заблокировать. Например:
```
A2:C6
C2:B2
```
9. Режим белого списка.
В **.env** файле выствьте флаг **USE_WHITELIST** на **True**.

Откройте файл **whitelist.txt** и укажите последние 2 байта MAC-адреса нод, которые должны быть в белом списке. Например:
```
A2:C6
C2:B2
```

Далее включите сервер или добавьте его в автозагрузку.
